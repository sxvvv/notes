## 1. GPU 与 CPU 的设计差异
GPU最初为图形渲染设计，适合处理**大量独立的多边形（着色、纹理化等）**，而 CPU 设计用于**通用程序的逻辑控制**。

- GPU：拥有大量的处理单元（算术逻辑单元 ALU / 浮点单元 FPU）和高内存带宽，但控制逻辑简单、缓存较小。

- CPU：拥有复杂的指令控制单元、大容量缓存，适合复杂的工作流和串行任务，但计算单元数量少。

## 2. 性能对比：GPU vs CPU

## 3. 异构应用程序

异构系统将高度并行的任务卸载到 GPU，而 CPU 处理控制密集部分，充分发挥两者优势。
GPU 可以是普通显卡（如 Quadro）或专用加速卡（如 Tesla），后者针对 HPC(High-Performance Computing，高性能计算) 优化，具备更强的双精度能力、特殊内存等。

## 4. 线程和核心的重新定义

| GPU 术语 | GPU 中的含义 | CPU 类比 |
| --- | --- | --- |
| thread (线程) | 分配给一个 CUDA 核心的指令流；多个线程执行同一条指令处理不同数据（SIMT） | N/A |
| CUDA core | 处理单个数据项的执行单元，属于 SIMT 指令流的一部分 | 向量通道 (vector lane) |
| warp (线程束) | 32 个线程的集合，它们在同一时间执行同一条指令（处理不同数据） | 向量 (vector) |
| kernel (内核) | 在设备上运行的函数，可划分为多个线程块 | 线程 (thread) |
| SM (流多处理器) | 执行一个线程块的硬件单元；多个 SM 可共同执行一个内核 | 核心 (core) |

一个 GPU 中的 CUDA 核心更像 CPU 向量处理单元中的一个通道，而不是完整的 CPU 核心。
GPU 以 32 个核心为一组（warp）执行 SIMD(Single Instruction, Multiple Data，单指令多数据) 的并行。

## 5. SIMT 与 Warp

### 5.1 SIMT（单指令多线程）

SIMT 是 NVIDIA 提出的术语，与 SIMD 类似但更灵活：允许线程根据条件激活或停用，从而支持分支（尽管效率不高）。

- 纯 SIMD：一条指令以完全相同方式作用于所有数据。

- SIMT：可对部分线程“掩蔽”，使分支中的不同路径依次执行，非活跃线程等待。这样保证了功能正确性，但活跃线程闲置时浪费计算能力。

### 5.2 Warp（线程束）

- 定义：由 32 个连续索引的线程组成的集合，它们在硬件上被一起调度执行。

- 原因：NVIDIA GPU 的 CUDA 核心以 32 个为一组（类似于 CPU 的向量处理单元），将线程块分割为 warp 简化了 SM 的调度。

- 性能影响：若同一个 warp 内的线程分支到不同路径，会导致串行化执行，降低效率。因此编程时应尽量避免 warp 内的分支。

## 6. Kernel 与 SM

### 6.1 Kernel（内核）

- 定义：在 GPU 上并行执行的函数，用 __global__ 修饰，可被主机或设备调用。

- 执行方式：由 N 个 GPU 线程并行执行 N 次，每个线程拥有唯一 ID，用于数据索引和决策。

### 6.2 SM（流多处理器）

- 作用：执行线程块的硬件单元。每个 SM 包含多组 CUDA 核心（每组 32 个）、调度器、寄存器、共享内存等。

- 分配策略：一个线程块只能分配给一个 SM，不能拆分。若块数多于 SM 数，则多个块可能分配给同一 SM（由硬件调度）。

- 内部执行：SM 将块内的线程划分为 warp，每个周期调度器分配 warp 到空闲的 CUDA 核心组执行。

### 6.3 内存层次
SM 内部包含：

- 寄存器

- L1 缓存

- 常量缓存

- 共享内存（可被同一块内所有线程访问）

- 全局内存（DRAM）则被所有 SM 共享。

## 7. 计算能力

- 定义：NVIDIA GPU 的架构功能版本号，定义了硬件支持的指令集、存储结构、数据类型与执行模型，是 CUDA 程序编译与兼容性的核心依据。
  
- 格式：主版本号.次版本号，如 7.0（Volta）、7.5（Turing）、8.0（Ampere）。主版本相同表示核心架构相同，次版本为增量改进。

- 用途：开发时可查询设备的计算能力以使用对应的特性，运行时也可动态判断。
