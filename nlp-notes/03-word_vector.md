# 词向量表示

## 一、为什么需要词向量？

以**文本分类**任务为例：假设我们要判断"这家火锅太好吃了"是正面评价还是负面评价。
人类一眼就能理解，但对计算机来说，不管是分词后的 `["这家", "火锅", "太", "好吃", "了"]`，还是单个汉字，本质上都只是无法直接理解的"字符串"。

机器学习模型处理的输入必须是**数字**——准确地说，是由数字组成的**向量或矩阵**。

所以我们的核心任务是：**在自然语言（符号世界）和数学模型（向量空间）之间架起一座桥梁。** 
这个将文字转换为数字的过程，统称为**词向量表示（Word Representation）**。

> **词嵌入（Word Embedding）** 是词向量家族中的一员，特指由神经网络学习得到的稠密、低维的词向量。

仅仅把词变成数字还不够。一个好的词向量，不仅要能区分每一个词，更理想的是向量本身能**蕴含语义信息**。
比如，我们希望"咖啡"和"奶茶"的向量距离很近，而"咖啡"和"拖拉机"的距离很远。

## 二、离散表示

在深度学习普及之前，研究者们提出了多种将词语表示为向量的方法。
这些方法将每个词视为独立单元，生成的向量维度高且稀疏，被称为**离散向量**。

### 2.1 独热编码（One-Hot Encoding）

#### 编码原理

独热编码是最直观的词表示方法。步骤很简单：

1. **构建词典**：收集语料中所有不重复的词
2. **分配索引**：给每个词一个从 0 开始的编号
3. **创建向量**：用一个长度等于词典大小的向量表示每个词，该词对应位置为 `1`，其余为 `0`

例如，从句子"我喜欢喝咖啡"构建词典 `["我", "喜欢", "喝", "咖啡"]`，各词的独热向量为：

```
"我"   -> [1, 0, 0, 0]
"喜欢" -> [0, 1, 0, 0]
"喝"   -> [0, 0, 1, 0]
"咖啡" -> [0, 0, 0, 1]
```

#### 优点与缺陷

**优点**：实现简单，能清晰地区分每个词。

**缺陷**：

- **维度灾难**：词典有 5 万个词，每个向量就是 5 万维，极度浪费资源
- **语义鸿沟**：任意两个不同词的向量**点积为 0**（正交），模型无法知道"咖啡"和"奶茶"比"咖啡"和"拖拉机"更像

> **为什么点积为 0？** 独热向量中每个词只有一个位置是 `1`，其余全是 `0`。两个不同的词，`1` 的位置必然错开，所以对应元素相乘全是 0，加起来还是 0。点积为 0 在几何上意味着两个向量"互相垂直"——模型认为所有词都同等不相关。

### 2.2 词袋模型（Bag-of-Words, BoW）

独热编码表示的是单个词，但实际中我们经常需要表示整个句子或文档。**词袋模型**就是为此而生的。

#### 基本思想

词袋模型的核心思想：**忽略词序，只关注"袋子里有什么词、各出现了几次"。**

实现上，相当于把文档中所有词的独热向量相加。假设词典是 `["我", "喜欢", "喝", "咖啡", "奶茶", "也"]`，现在有两段评论：

- 评论 A：`我 喜欢 喝 咖啡`
- 评论 B：`我 也 喜欢 喝 奶茶`

它们的词袋表示：

```
vec(评论A) = [1, 1, 1, 1, 0, 0]   （"我"1次、"喜欢"1次、"喝"1次、"咖啡"1次）
vec(评论B) = [1, 1, 1, 0, 1, 1]   （"我"1次、"喜欢"1次、"喝"1次、"奶茶"1次、"也"1次）
```

通过计算向量距离，我们能发现这两条评论很相似——它们共享了"我"、"喜欢"、"喝"三个词。

#### 余弦相似度计算

**余弦相似度**通过计算两个向量夹角的余弦值来衡量相似性，值越接近 1 越相似：

$$\text{similarity} = \cos(\theta) = \frac{\mathbf{A} \cdot \mathbf{B}}{||\mathbf{A}|| \cdot ||\mathbf{B}||} = \frac{\sum_{i=1}^{n} A_i B_i}{\sqrt{\sum_{i=1}^{n} A_i^2} \cdot \sqrt{\sum_{i=1}^{n} B_i^2}} $$

用上面的例子算一下：令 **A** = `[1, 1, 1, 1, 0, 0]`，**B** = `[1, 1, 1, 0, 1, 1]`

- **点积**：

  $$1 \times 1 + 1 \times 1 + 1 \times 1 + 1 \times 0 + 0 \times 1 + 0 \times 1 = 3$$

- **模长**：


  $|\mathbf{A}| = \sqrt{1+1+1+1+0+0} = 2$，

  $|\mathbf{B}| = \sqrt{1+1+1+0+1+1} = \sqrt{5}$


- **相似度**：

  $$\cos(\theta) = \frac{3}{2 \times \sqrt{5}} \approx 0.67$$

结果 0.67，说明这两条评论确实比较相似。

#### 不同统计方式

词袋向量中每一维的值可以有不同选择：**频数**（出现次数）、**频率**（次数除以总词数，消除长文档偏差）、**二进制**（只关心出不出现，不管几次）。

**优点**：实现简单，在文本分类等任务上效果不错。

**缺点**：

- **丢失词序**："我 喜欢 你"和"你 喜欢 我"的词袋表示完全相同
- **未考虑词的重要性**："的"、"是"这类停用词频次极高，却对区分文档主题毫无帮助

### 2.3 TF-IDF

为了解决"常见词权重过高"的问题，**TF-IDF** 被提出。

> **核心理念**：一个词在**当前文档**中出现越多（TF 高），同时在**其他文档**中出现越少（IDF 高），说明它对当前文档越重要。

**TF-IDF 由两部分组成**：

**词频（TF）**：衡量一个词在当前文档中出现的频繁程度。

$$TF(t, d) = \frac{f_{t,d}}{\sum_{t' \in d} f_{t',d}}$$

**逆文档频率（IDF）**：衡量一个词的"稀有"程度。一个词出现在越多文档中，它的 IDF 值越低。

$$IDF(t, D) = \log \frac{|D|}{1 + |{d \in D : t \in d}|} $$

最终权重就是二者的乘积：

$$TF\text{-}IDF(t, d, D) = TF(t, d) \times IDF(t, D) $$

> **直观理解**：假设你在分析一篇关于"量子计算"的论文。"的"、"是"这些词虽然在文中出现很多次（TF 高），但几乎所有文档里都有（IDF 很低），所以 TF-IDF 值很低。而"量子比特"可能只在少数专业文档中出现（IDF 高），在这篇论文中又频繁出现（TF 高），因此 TF-IDF 值很高——说明它是关键词。

**实际应用**：关键词提取（`jieba` 就内置了此功能）、文本相似度计算、传统搜索引擎排序。

### 2.4 N-gram 模型

词袋模型和 TF-IDF 虽然有效，但它们都**彻底丢失了词序信息**。**N-gram** 通过统计连续词组来弥补这一短板，同时它也是现代大语言模型的**鼻祖**——因为它最早引入了**预测下一个词**的思想。

#### 预测下一个词

N-gram 的核心基于**马尔可夫假设**：一个词出现的概率只取决于它前面的 N-1 个词。

| 类型 | 依赖前文 | 示例（句子："我喜欢吃火锅"） |
|------|---------|---------------------------|
| Unigram (1-gram) | 无依赖 | 每个词独立，和词袋一样 |
| Bigram (2-gram) | 前 1 个词 | `{"我 喜欢", "喜欢 吃", "吃 火锅"}` |
| Trigram (3-gram) | 前 2 个词 | `{"我 喜欢 吃", "喜欢 吃 火锅"}` |

通过 Bigram，模型就能区分"我 喜欢"和"喜欢 我"了——它们是完全不同的词组特征。

> **和今天的 GPT 有什么关系？** 现在的大语言模型本质上可以看作一个 N 极其巨大（比如 N=128000）的超级 N-gram 模型——它们都在做"根据上文预测下一个词"。

#### N-gram 的挑战

N-gram 找回了词序，但代价巨大：

- **指数级爆炸**：词典有 1 万个词，Bigram 就有 $10^8$ 种组合，Trigram 高达 $10^{12}$
- **数据稀疏**：绝大多数词组（如"仙人掌 驾驶 潜水艇"）在真实语料中永远不会出现，概率为 0

正是为了从根本上解决这些问题，后来才诞生了词向量和神经网络语言模型。

## 三、序号化表示

在深度学习时代，上面那些精巧的特征工程不再是主流。现代模型采用一种更简洁的方式——**序号化**：只做最少的预处理，把"学习词语含义"这个复杂任务交给模型自己完成。

### 3.1 序号化过程

序号化就是把分词后的词元序列转换为整数序列，分三步：

**第一步：构建词典。** 与独热编码类似，但在深度学习中通常是字级别（如 BERT）或子词级别（如 GPT）的词典。

**第二步：增加特殊词元。** 在词典中加入功能性 Token：

| 特殊词元 | 通常 ID | 作用 |
|---------|--------|------|
| `[PAD]` | 0 | 填充短句子，使同一批次的句子长度一致 |
| `[UNK]` | 1 | 代替词典中没有的陌生词 |
| `[CLS]` | 视模型而定 | 分类任务的标记（BERT 使用） |
| `[SEP]` | 视模型而定 | 句子分隔符 |

**第三步：ID 映射。** 把每个词元直接替换为它在词典中的整数编号。

> 实际中，很少自己从零建词典，通常直接使用预训练模型（BERT、GPT 等）自带的词典文件 `vocab.txt`。

### 3.2 序号化实例

假设有三个句子需要处理，使用一个精简的字级别词典：

```
{'[PAD]': 0, '[UNK]': 1, '我': 2, '喜': 3, '欢': 4, '喝': 5, '咖': 6, '啡': 7, '奶': 8, '茶': 9, '也': 10}
```

**第一步：分词 + ID 映射**

```
句子1（我喜欢喝咖啡）：我(2) 喜(3) 欢(4) 喝(5) 咖(6) 啡(7)  → [2, 3, 4, 5, 6, 7]
句子2（我也喝茶）：    我(2) 也(10) 喝(5) 茶(9)             → [2, 10, 5, 9]
句子3（我喝水）：      我(2) 喝(5) 水(不在词典中→UNK=1)      → [2, 5, 1]
```

**第二步：填充（Padding）**

以最长的句子 1（长度 6）为基准，用 `[PAD]=0` 填充短句子：

```
句子1：[2, 3, 4, 5, 6, 7]
句子2：[2, 10, 5, 9, 0, 0]
句子3：[2, 5, 1, 0, 0, 0]
```

**最终输入模型的张量（Tensor）**：

```
[[2,  3, 4, 5, 6, 7],
 [2, 10, 5, 9, 0, 0],
 [2,  5, 1, 0, 0, 0]]
```

> **关键理解**：序号化本身并没有解决语义问题——ID `3` 和 `4` 之间没有数学意义上的关系。它的真正价值是作为后续**嵌入层（Embedding Layer）** 的输入。嵌入层会把这些 ID 查表映射为低维、稠密的浮点数向量（即词向量），而这个映射关系是在模型训练中**学习**出来的。
