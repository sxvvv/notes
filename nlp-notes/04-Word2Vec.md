# Word2Vec

## 一、寻找理想的词向量

无论是独热编码还是序号化，它们都有一个根本缺陷：**无法表达词与词之间的语义关系。** 在独热编码中，所有词两两正交；在序号化中，ID 的大小关系纯属随机——模型无法理解"猫"和"狗"比"猫"和"飞机"语义更近。

为了解决这个问题，**分布式表示（Distributed Representation）** 被提出，目标是将词语映射到一个低维、稠密、蕴含语义的连续向量空间。

理想的词向量需要满足两个条件：

- **语义蕴含**：语义相近的词，向量距离也近。比如"猫"和"狗"的向量应该靠在一起，"猫"和"火箭"应该远离。这背后的原理是**分布式假设**——如果两个词经常出现在相似的上下文中，它们的语义就相近。
- **低维稠密**：向量维度是一个可控的小数字（如 128 或 300），而不是动辄数万的词典大小；每一维都是有意义的浮点数，而非绝大部分为 0 的稀疏表示。

实现这一目标有两条技术路线：基于**全局文档统计**的主题模型，和基于**局部上下文预测**的神经网络模型（Word2Vec）。

---

## 二、主题模型

主题模型从宏观视角出发，通过分析大量文档中词语的共现统计，来发现潜在的语义关联。

它的关键假设是：**一篇文档由多个"主题"混合而成，一个主题又由多个"词语"按概率组成。** 词语之所以会在同一篇文档中出现，是因为它们在共同描述某个主题。

> **举个例子**：一篇美食博客里频繁出现"面粉"、"烤箱"、"黄油"、"发酵"这些词。它们之所以共现，是因为都在描述"烘焙"这个主题。所以，一个词的向量就可以用它与各个主题的关联强度来表示。

核心技术是**矩阵分解（Matrix Factorization）**。

### 2.1 SVD 矩阵分解

将获取词向量的过程转化为矩阵分解问题，分三步：

**第一步：构建"词-文档"矩阵**

以整个语料库为基础，构建一个大矩阵 $X$：行是词、列是文档，每个格子填入该词在该文档中的 TF-IDF 权重。这个矩阵通常又大又稀疏。

> **直观理解**：想象一张巨大的 Excel 表格，每一行是一个词（比如"面粉"、"GPU"、"进球"），每一列是一篇文档（美食博客、技术论文、体育新闻），表格里填的是每个词对每篇文档有多重要。

**第二步：矩阵分解（SVD）**

把这个巨大的稀疏矩阵近似分解为两个小矩阵的乘积：

$$X_{m \times n} \approx W_{topic, m \times k} \times H_{topic, k \times n} $$

其中：

- $m$ = 词典大小，$n$ = 文档数量
- $k$ = 潜在主题数量（远小于 $m$ 和 $n$ 的超参数，比如设为 100）
- $W_{topic}$（词-主题矩阵）：每一行是一个词的 $k$ 维向量，表示该词与各个主题的关联度
- $H_{topic}$（文档-主题矩阵）：每一列是一篇文档的 $k$ 维向量，表示该文档在各个主题上的分布

**第三步：获取词向量**

$W_{topic}$ 的每一行就是我们要的词向量！它把原来 $m$ 维的独热编码压缩到了 $k$ 维的稠密向量。

> **为什么能捕捉语义？** 如果"面粉"和"黄油"经常在描述"烘焙"主题的文档中共同出现，SVD 分解后它们在"烘焙"这个主题维度上都会有很高的值，从而使它们的词向量在空间上非常接近。

### 2.2 主题模型的局限性

从机器学习角度看，主题模型本质上是一个**聚类算法**——把文档聚成若干个主题，同时揭示每个词与各主题的关联。它在一定程度上缓解了同义词问题：虽然"番茄"和"西红柿"写法不同，但因为都高频出现在"烹饪"相关文档中，它们的向量会很相似。

但主题模型（也叫 **LSA, Latent Semantic Analysis**）有明显的局限：

- **计算代价高**：对大型语料库做 SVD 分解，计算量和内存开销极大
- **信息粒度粗**：只用了文档级别的共现信息，忽略了词在句子中的局部上下文和词序
- **难以集成**："先统计再分解"的流程很难与深度学习模型进行端到端的联合训练

## 三、Word2Vec

与主题模型从全局统计出发不同，Google 在 2013 年提出的 **Word2Vec** 将视角聚焦于词语的**局部上下文**。

它背后的思想来自语言学中的**分布式假设**：

> **一个词的含义，由它周围的词所决定。**

如果两个词的上下文经常是相似的，那么它们的语义就相近。Word2Vec 就是这一思想的数学实现。

### 3.1 Word2Vec 概述

Word2Vec 是一种**浅层神经网络模型**——它移除了传统模型中计算昂贵的非线性隐藏层，结构非常简洁，因此能在大规模语料上高效训练。

理解 Word2Vec 的关键在于区分**目标**和**手段**：

- **最终目标**：获取一个高质量的**词向量查询表**（一个大矩阵 $W_{in}$，每行是一个词的稠密向量）
- **实现手段**：设计一个"伪任务"（根据上下文预测中心词，或反之），在训练过程中优化这个查询表
- **训练完成后**：执行预测的网络被丢弃，我们只保留那个**词向量查询表**

> **打个比方**：就像你为了练习英语而去参加英语辩论赛。辩论赛（伪任务）本身不是目的，你的英语能力提升（词向量质量）才是。比赛结束后，奖杯可以扔掉，但英语能力留下了。

### 3.2 可学习的词向量矩阵

从数学上看，把一个单词 ID 转换为稠密向量的过程可以分三步理解：

1. **输入**：单词 ID（例如 3）
2. **独热编码**：变成 $[0, 0, 0, 1, 0, \ldots]$（只有第 3 位是 1）
3. **矩阵乘法**：用这个独热向量乘以可学习的参数矩阵 $W_{in}$（大小 $|V| \times D$）

由于独热向量只有一个位置是 1，矩阵乘法的结果就等于直接**取出 $W_{in}$ 的第 3 行**。

> **实际实现中**：程序不会真的做稀疏矩阵乘法，而是直接根据 ID 去查表取行——这就是 PyTorch 中 `nn.Embedding` 层在做的事。

关键在于，这个矩阵 $W_{in}$ **本身就是学习的目标**。它被随机初始化，然后通过 CBOW 或 Skip-gram 任务在训练过程中不断被优化。

### 3.3 模型架构与实现原理

#### 3.3.1 两种经典模型

Word2Vec 有两种实现方式，它们的任务设计恰好相反：

---

**（1）CBOW 模型：根据上下文预测中心词**

CBOW（Continuous Bag-of-Words）的任务是：给定一个词的周围词（上下文），预测中间那个词（中心词）。

**用一个生活化的例子来理解**：

假设句子是"早上 我 去 咖啡店 买了 一杯 拿铁"，窗口大小为 2（中心词左右各取 2 个词）。

当中心词是"咖啡店"时：

- 上下文 = `["我", "去", "买了", "一杯"]`
- 任务 = 根据这 4 个词，预测出"咖啡店"

**数学流程**：

1. **词向量转换**：从 $W_{in}$ 中查出上下文每个词的向量
2. **上下文聚合**：将这些向量求平均，得到一个上下文向量 $h$
3. **输出得分**：将 $h$ 与输出矩阵 $W_{out}$ 相乘，得到词典中每个词的预测得分
4. **损失计算**：通过 Softmax 转成概率分布，与真实中心词计算交叉熵损失

损失函数：

$$J = -\log P(w_c | \text{上下文}) = -\log \frac{\exp(u_c^T h)}{\sum_{j=1}^{|V|} \exp(u_j^T h)} $$

其中 $u_c$ 是目标中心词的输出向量，$h$ 是上下文向量。

**具体数值跟踪**（帮助建立直觉）：

假设批大小 $B=2$，上下文窗口大小 $S=4$（左右各 2），词典大小 $|V|=8000$，词向量维度 $D=100$：

| 步骤 | 操作 | 数据形状 |
|-----|------|---------|
| 输入 | 上下文词的 ID | $(2, 4)$ |
| 查表 | 每个 ID 变成 100 维向量 | $(2, 4, 100)$ |
| 求平均 | 4 个向量压缩为 1 个 | $(2, 100)$ |
| 输出得分 | 乘以 $W_{out}$ | $(2, 8000)$ |
| 损失 | Softmax + 交叉熵 | 标量 |

---

**（2）Skip-gram 模型：根据中心词预测上下文**

Skip-gram 与 CBOW 恰好相反：给定中心词，预测它周围的词。

同样的例子："早上 我 去 咖啡店 买了 一杯 拿铁"，中心词是"咖啡店"：

- 输入 = `"咖啡店"`
- 目标 = 分别预测 `"我"`、`"去"`、`"买了"`、`"一杯"`

它把一个预测任务拆成了多个独立的子任务。

损失函数：

$$J = -\sum_{j=0, j \neq m}^{2m} \log \frac{\exp(u_{c-m+j}^T v_c)}{\sum_{k=1}^{|V|} \exp(u_k^T v_c)} $$

其中 $v_c$ 是中心词的输入向量，$u_{c-m+j}$ 是各上下文词的输出向量。

| 步骤 | 操作 | 数据形状 |
|-----|------|---------|
| 输入 | 中心词 ID | $(2, 1)$ |
| 查表 | 变成 100 维向量 | $(2, 1, 100)$ |
| 输出得分 | 乘以 $W_{out}$ | $(2, 8000)$ |
| 损失 | 得分被复用 4 次，分别与 4 个上下文词计算交叉熵并求和 | 标量 |

> **Skip-gram vs CBOW**：Skip-gram 为每个"中心词-上下文词"对都创建了独立的学习任务，因此在处理**低频词**和**大数据集**时通常能得到更好的词向量，但训练速度较慢。

> **Skip-gram 和 N-gram 是什么关系？** 名字来源于传统的 k-skip-n-gram（允许跳过中间词的 N-gram），但 Word2Vec 的 Skip-gram 是**预测模型**而非统计模型——它通过预测任务强迫模型学习语义特征，从根本上解决了 N-gram 的稀疏性和维度灾难问题。

#### 3.3.2 滑动窗口：语义是怎么学到的？

了解了模型结构后，一个自然的问题是：**模型是怎么真正学到"猫"和"狗"语义相近的？**

秘密在于**滑动窗口**生成的大量高度重叠的训练样本。

以 CBOW 为例，假设句子很长，窗口大小 $k=3$（中心词左右各 3 个词）：

- 窗口 1：上下文 = `[..., 小猫, 在, 沙发]` + `[睡觉, ...]` → 预测"上"
- 窗口 2：上下文 = `[..., 小狗, 在, 沙发]` + `[睡觉, ...]` → 预测"上"

这两个窗口的上下文高度相似（都包含"在"、"沙发"、"睡觉"），而且可能预测的目标词也相同。

**模型面临的"矛盾"**：几乎相同的上下文要能预测出正确结果。模型发现的"捷径"是——**让"小猫"和"小狗"的词向量本身就足够接近**，这样相似的上下文向量就能同时很好地预测出正确目标。

从数学角度看，模型要最大化的得分是上下文向量与目标词向量的**点积**：

$$\text{score} = x \cdot u_{target} $$

最大化点积在几何上就是让两个向量的夹角尽可能小——即在空间上更接近。

这个过程在整个语料库中不断重复：当两个词（如"咖啡"和"奶茶"）因为语言习惯而频繁出现在相似的上下文中（都和"喝"、"杯"、"店"共现），模型就会把它们的词向量推向彼此靠近的位置。

> 实际训练中，为避免对整个词表做 Softmax 的高开销，通常使用 **Hierarchical Softmax** 或**负采样（Negative Sampling）** 来加速。

## 四、Word2Vec 的局限

尽管 Word2Vec 是里程碑式的算法，但它有一个根本性的局限：**静态词向量**。

**上下文无关**：对于词典中的任何一个词，Word2Vec 只生成**一个固定的**向量。这个向量是在整个语料库上训练得到的"平均"语义，与具体上下文无关。

> **这意味着什么？** "苹果"这个词，无论出现在"我今天吃了一个苹果"还是"苹果公司发布了新产品"中，Word2Vec 给它的向量都**完全相同**。模型无法区分一词多义。

**静态的本质**：Word2Vec 的输出就是一个固定的查询表。使用时只是根据词 ID 查找对应的行向量，整个过程不涉及任何对上下文的动态分析。

> 后来的 ELMo、BERT、GPT 等模型正是为了解决这个问题而诞生的——它们生成的是**动态词向量**，同一个词在不同语境下会得到不同的向量表示。
