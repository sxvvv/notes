# 第一节 初级分词技术

要让计算机理解人类语言，第一步就是把连续的文本"切碎"成一块块有意义的小单元——这个过程叫做**分词（Tokenization）**。

打个比方：文本就像一整条长面包，而"词"就是切好的面包片。分词的好坏，直接决定了后续任务（如搜索、翻译、情感分析等）的效果。

---

## 一、分词的定义与重要性

**分词**的任务，是把连续的文本序列切分成具有独立语义的基本单元（即"词"或"词元"）。

对于英语来说，单词之间天然有空格分隔，分词比较简单。但对于中文、日文、泰文等语言，文本是连续书写的，词之间没有明确的边界。

例如，对于句子 `"今天天气真不错"`，计算机需要依据算法将其正确切分为：

```
["今天", "天气", "真", "不错"]
```

在传统的 NLP 处理流程中，分词是所有后续任务的基础。如果分词出错，这个错误会像"多米诺骨牌"一样一路传递下去，产生**级联效应（Cascading Effect）**。

例如，在搜索引擎中，如果把"南京市长江大桥"错误地分成了 `["南京", "市长", "江大桥"]`，系统就会误以为用户在搜索某位"市长"，而不是一座"大桥"，搜索结果就完全跑偏了。

现代 NLP 方法通过更灵活的切分策略，在很大程度上缓解了这个问题。

## 二、通过 jieba 认识分词

虽然在以 Transformer 为基础的大语言模型兴起后，传统"语言学分词"的应用场景逐步减少，现代大语言模型更多采用子词（Subword）策略。但 `jieba` 作为一个经典、简单、开源的中文分词库，仍然是**理解分词思想、学习 NLP 基础**的绝佳工具。

### 2.1 jieba 安装

`jieba` 是目前流行的 Python 中文分词库之一，安装非常简单：

```bash
pip install jieba
```

安装完成后，可以用以下命令确认安装是否成功：

```bash
pip show jieba
```

### 2.2 基于规则与词典

**基于规则与词典**的分词方法是 NLP 领域最早期的技术，也是最符合直觉的方式。核心依赖两样东西：**一部大型词典** 和 **一套匹配规则**。

`jieba` 的默认模式就是这种方式。它的工作原理可以分为两步：

1. 基于一个**前缀词典**，找出句子中所有可能的词语组合，构建一个**有向无环图（DAG）**
2. 用**动态规划**算法从中找出概率最大的那条路径，作为最终分词结果

用数学来表达：假设一种分词方案把句子切成了词语序列 $w_1, w_2, ..., w_n$，那么这条路径的概率近似为：

$$P(w_1, w_2, ..., w_n) \approx P(w_1) \times P(w_2) \times ... \times P(w_n) \tag{2.1}$$

其中，每个词 $w_i$ 的概率可以通过它在词典中出现的频率来估算：

$$P(w_i) = \frac{\text{词 } w_i \text{ 的词频}}{\text{词典中所有词的总词频}} \tag{2.2}$$

`jieba` 的目标就是找到一条路径，使这个累乘的概率值最大。

#### 2.2.1 Log 概率与动态规划

在实际计算中，把一堆小于 1 的数连续相乘，结果会变得极其微小，导致计算机无法精确表示——这就是**浮点数下溢**问题。

解决办法是使用**对数概率（Log 概率）**：利用对数的性质，把概率的"乘法"变成对数概率的"加法"。找概率最大值，就等价于找对数概率之和的最大值：

$$\underset{w}{\arg\max} \sum_{i=1}^{n} \log P(w_i) \tag{2.3}$$

> **初学者理解要点**：`log` 函数是单调递增的，所以"谁的 log 值更大，谁的原始值也更大"。把乘法变成加法，计算更稳定，结果完全一致。

接下来，为了避免穷举所有路径带来的巨大计算量，`jieba` 使用**动态规划**：从句子末尾开始，逐步向前计算每个位置的最优切分，并记录下来。最终从句首出发，沿着记录好的路径就能得到全局最优的分词结果。

**举个例子**：对于句子"今天天气真不错"，在"天气"这个位置可能存在两条候选路径：

- **路径 A**：`今天 / 天气 / 真 / 不错`（把"天气"当作一个词）
- **路径 B**：`今天 / 天 / 气 / 真 / 不错`（把"天"和"气"拆开）

用对数概率计算：

$$\log P(A) = \log P(\text{今天}) + \log P(\text{天气}) + \log P(\text{真}) + \log P(\text{不错})$$

$$\log P(B) = \log P(\text{今天}) + \log P(\text{天}) + \log P(\text{气}) + \log P(\text{真}) + \log P(\text{不错})$$

由于"天气"是一个高频词，它的整体概率远大于"天"和"气"两个单字概率的乘积，因此：

$$\log P(\text{天气}) \gg \log P(\text{天}) + \log P(\text{气}) \tag{2.4}$$

所以：

$$\log P(A) > \log P(B) \tag{2.5}$$

动态规划会自动在每个位置完成这种比较，最终选出全局最优路径 A。

#### 2.2.2 Jieba 实践

前面介绍的方法，在 `jieba` 中对应的是**精确模式**。下面用代码来看看效果：

```python
import jieba

text = "今天天气真不错我们去公园吧"
seg_list = jieba.lcut(text, cut_all=False)  # cut_all=False 表示精确模式
print(seg_list)
```

输出：

```
['今天', '天气', '真', '不错', '我们', '去', '公园', '吧']
```

上面的例子之所以切得理想，是因为"天气"、"公园"等词在词典中已有记录。但如果遇到词典里没有的新词（比如"冰墩墩"这类新造词），就容易被拆散成单字。

这时候，**自定义词典**就派上用场了——我们可以手动"教会" `jieba` 认识新词。创建一个 `user_dict.txt` 文件：

```
冰墩墩
雪容融
```

> `jieba` 自定义词典的格式为 `词语 [词频] [词性]`，词频和词性可选。

然后运行代码，观察加载词典前后的效果变化：

```python
# 未加载词典前
text = "冰墩墩和雪容融是北京冬奥会的吉祥物"
print(f"精确模式: {jieba.lcut(text, cut_all=False)}")

# 加载自定义词典
jieba.load_userdict("./user_dict.txt")
print(f"加载词典后: {jieba.lcut(text, cut_all=False)}")
```

输出：

```
精确模式: ['冰', '墩', '墩', '和', '雪', '容', '融', '是', '北京', '冬奥会', '的', '吉祥物']
加载词典后: ['冰墩墩', '和', '雪容融', '是', '北京', '冬奥会', '的', '吉祥物']
```

通过自定义词典，能方便地解决 **OOV（未登录词）** 问题，在处理特定领域文本时尤其有用。

> **什么是 OOV？** 未登录词（Out-Of-Vocabulary）是一个相对概念——指某个词典中未收录的词。典型表现是：本该作为一个整体的词，因为不在词典中而被错误地切碎成单字。

#### 2.2.3 精确模式工作流程

`jieba` 精确模式的完整工作流程分为四步（以句子"今天天气真不错"为例）：

**第一步：文本预处理与分块**

利用正则表达式把输入文本分成"汉字区块"和"非汉字区块"（标点、数字、英文等）。非汉字部分直接输出，汉字部分进入后续分词流程。

**第二步：构建有向无环图（DAG）**

扫描句子中的每个字，查词典看它能组成哪些词。例如：

- 从"今"开始：能组成"今"、"今天"
- 从"天"开始：能组成"天"、"天气"
- 从"气"开始：能组成"气"
- ...

这些字与词之间的连接关系就构成了一张"地图"——有向无环图。

**第三步：动态规划计算最优路径**

从句子末尾向前逐步计算：每个位置选择哪个词能让总概率最大。例如先看"错"，再看"不"（是切成"不错"还是"不"+"错"哪个概率高？），以此类推直到句首。

**第四步：根据路由表输出结果**

沿着记录好的最优路径，从头到尾依次取出词语，得到最终结果：`['今天', '天气', '真', '不错']`。

### 2.3 统计学习时代的方法

为了解决对人工词典的过度依赖，研究者们转向了统计学习。核心思想是把分词看作一个**序列标注**问题——为句子中的每个字打上一个位置标签：

| 标签 | 含义 | 示例 |
|------|------|------|
| B | 词的开头 | "天气"的"天" |
| M | 词的中间 | "巧克力"的"克" |
| E | 词的结尾 | "天气"的"气" |
| S | 单字成词 | "真"、"去" |

例如"我爱北京"会被标注为 `S S B E`（"我"单字成词、"爱"单字成词、"北"是词首、"京"是词尾）。

**隐马尔可夫模型（HMM）** 就是解决这类序列标注问题的经典模型。`jieba` 利用 HMM 来识别词典中没有的新词。当基于词典的方法遇到无法切分的连续字串时，就会调用 HMM 进行二次分词。

#### 2.3.1 HMM 对未登录词的识别

纯词典模式遇到未收录的词时，只能把它切碎成单字。而默认开启 HMM 的模式则采用了**"先攒后处理"**的策略：

1. 遍历动态规划的分词路径
2. 遇到单字先放进"缓冲区"
3. 一旦遇到多字词或句末，就把缓冲区里攒的字交给 HMM 进行二次分词
4. HMM 根据学到的统计规律，尝试把被切碎的字重新"粘"成词

看看实际效果：

```python
text = "小红书上有很多美食教程"

# 开启 HMM（默认）
seg_list_hmm = jieba.lcut(text, HMM=True)
print(f"HMM 开启: {seg_list_hmm}")

# 关闭 HMM
seg_list_no_hmm = jieba.lcut(text, HMM=False)
print(f"HMM 关闭: {seg_list_no_hmm}")
```

```
HMM 开启: ['小红书', '上', '有', '很多', '美食', '教程']
HMM 关闭: ['小红', '书上', '有', '很多', '美食', '教程']
```

可以看到，开启 HMM 后，即使"小红书"不在词典中，模型也能通过统计规律将它正确识别为一个完整的词。

#### 2.3.2 词性标注

除了分词，`jieba` 还能进行词性标注——识别每个词是名词、动词还是形容词等。需要用到 `jieba.posseg` 模块：

```python
import jieba.posseg as pseg

text = "冰墩墩和雪容融是北京冬奥会的吉祥物"

words = pseg.lcut(text, HMM=False)
print(f"词性标注输出: {words}")
```

输出：

```
词性标注输出: [pair('冰墩墩', 'x'), pair('和', 'c'), pair('雪容融', 'x'), pair('是', 'v'), pair('北京', 'ns'), pair('冬奥会', 'j'), pair('的', 'uj'), pair('吉祥物', 'n')]
```

这里"冰墩墩"和"雪容融"被标注为 `x`（非语素字），因为自定义词典中没有指定它们的词性。如果我们在词典里加上词性信息：

```
冰墩墩 5 nr
雪容融 5 nr
```

重新加载后，词性标注就会变得准确。

常见词性标签含义如下：

| 标签 | 含义 | 标签 | 含义 |
|------|------|------|------|
| n | 名词 | nr | 人名 |
| ns | 地名 | nt | 机构团体 |
| v | 动词 | a | 形容词 |
| d | 副词 | m | 数词 |
| r | 代词 | p | 介词 |
| c | 连词 | u | 助词 |
| t | 时间词 | x | 非语素字 |

### 2.4 从"分词"到"分块"

随着深度学习——特别是 `BERT` 和 `GPT` 等大规模预训练模型——的兴起，传统的分词方式发生了重大变化。现代模型更倾向于使用数据驱动的策略，主要有两种流派：

#### （1）字粒度分词

以 `BERT` 为代表。处理中文时，直接把每个汉字视为一个独立的 Token。

**优点**：彻底解决了 OOV 问题，因为常用汉字数量有限，可以构建一个全覆盖的"字表"。

**缺点**：
- "博物馆"被拆成"博"、"物"、"馆"三个字后，模型需要额外学习它们的组合关系
- 序列变长，计算量增大

> **初学者理解**：就好比把一句话的每个字都拆开写在单独的卡片上。不会漏掉任何字，但失去了"词"的整体感。

#### （2）子词分词

以 `GPT` 系列为代表，采用 **BPE（Byte Pair Encoding）** 算法。

BPE 的核心思想很简单：从最小的字符单元开始，反复合并出现频率最高的相邻字符对，逐步构建出一个大小可控的词表。

**举个直观的例子**：假设训练语料中 "learn"、"learning"、"learned" 出现得非常频繁：
- 首先 "l" 和 "e" 经常相邻，合并为 "le"
- 然后 "le" 和 "a" 经常相邻，合并为 "lea"
- 接着 "lea" 和 "rn" 合并为 "learn"
- 最终 "learn" 会作为一个整体进入词表，而 "ing"、"ed" 等后缀也会成为独立的子词单元

这样，"learning" = "learn" + "ing"，"learned" = "learn" + "ed"，既保留了词根的语义，又能灵活组合。

**优点**：在词表大小和语义表达之间取得了出色平衡，高频词完整保留，低频词和新词被拆解为有意义的小单元。

**缺点**：切分结果基于统计频率而非语言学规则，可解释性较差；对训练语料依赖较强。

尽管如此，子词分词已经成为当前大语言模型的标准方案。
